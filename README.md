# Face Brightness Control

基于 Electron 的智能人脸检测自动亮度控制应用，再也不用害怕笔记本架高后被身后的同事窥屏了，同时也间接解决了老板从背后出现却没人提醒你的烦恼

## ✨ 功能特性

- 🎥 **实时人脸检测**：使用 face-api.js 进行实时摄像头人脸检测
- 💡 **智能亮度控制**：检测到指定数量人脸时自动降低屏幕亮度
- 🔔 **系统通知提醒**：支持系统通知、亮度调节或两者组合，可自定义通知内容
- 🧠 **人脸识别**：支持学习识别本人、可信任、不可信任三类人群
- 👁️ **正脸识别**：可选仅识别正脸，侧脸不触发，减少误报
- 🎯 **人脸数量阈值**：可自定义触发亮度调节的人脸数量（默认≥2人）
- 🔧 **多模型支持**：支持 TinyFaceDetector（快速）和 SSD MobileNet（准确）
- ⚙️ **灵活配置**：可调节检测间隔、灵敏度、亮度范围、识别阈值等参数
- 🖼️ **可视化检测**：实时显示检测到的人脸框和置信度
- 📸 **照片采集工具**：内置工具快速采集多角度人脸照片
- 🖥️ **跨平台支持**：支持 Windows 和 macOS
- 📦 **绿色便携**：无需安装额外依赖，开箱即用

## 📖 使用说明

### 基本流程

1. 启动应用后，允许摄像头权限
2. 应用自动加载人脸检测模型
3. 点击"启动检测"开始实时检测
4. 当检测到的人脸数量达到阈值时，自动降低亮度
5. 人脸消失后，自动恢复原始亮度

### 参数设置

#### 基本设置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 检测模型 | SSD MobileNet | 更准确但稍慢，可选 TinyFaceDetector（快速） |
| 检测间隔 | 500ms | 检测频率，值越小越频繁（更耗电）<br>⚠️ **启用高级设置时建议设为 1000ms** |
| 检测灵敏度 | 0.5（低） | 近距离检测，可调整为高/中/低 |
| 最低亮度 | 10% | 检测到人脸时的目标亮度 |
| 最高亮度 | 自动检测 | 无人脸时的亮度（启动时自动获取当前亮度） |
| 人脸数量阈值 | 2 | 触发亮度降低所需的最小人脸数量 |
| 处理方式 | 仅调节亮度 | 可选：仅调节亮度 / 仅系统通知 / 两者组合 |
| 通知内容 | 检测到异常活动 | 自定义通知文本，自动保存 |
| 仅识别正脸 | 关闭 | 开启后只有正脸触发，侧脸不触发（提高准确性） |
| 正脸严格度 | 适中（±20°） | 可选：宽松（±30°）/ 适中（±20°）/ 严格（±10°） |

#### 高级设置（人脸识别）

| 功能 | 说明 |
|------|------|
| 人脸分类 | 可学习识别本人、可信任、不可信任三类人群 |
| 识别阈值 | 可调节识别严格度（0.3-0.7），默认0.5 |
| 识别模式 | 不启用 / 除本人外 / 除本人及可信任外 / 仅针对不可信目标 |
| 照片采集 | 菜单 → 工具 → 人脸照片采集工具（Ctrl+T / Cmd+T） |

> ⚠️ **性能提示**：高级设置会进行人脸识别计算，相比基本模式更耗性能。  
> 建议配置：**将检测间隔调整为 1000ms（1秒）**，可大幅降低 CPU 占用，同时保持良好的检测效果。

> 📌 **识别准确度说明**（重要）：  
> 本应用使用的 face-api.js 库基于 FaceNet 模型，在通用场景下表现良好，但**存在客观局限性**：
> 
> **常见问题：**
> - 😕 **误识别**：可能将相似面孔混淆（如双胞胎、相似度高的陌生人）
> - 🔄 **重复检测**：同一人可能被识别为多张人脸（光线、角度变化）
> - 📸 **训练数据敏感**：识别准确度高度依赖训练照片的质量和多样性
> - 🌓 **环境影响**：光线变化、戴口罩、遮挡等都会降低准确度
> 
> **建议配置：**
> 1. ⚙️ **调整识别严格度（阈值）**：
>    - 默认 `0.5` 较宽松，可能误识别 → **建议调至 `0.4` 或 `0.35`**
>    - 阈值越小越严格，降低误识别率
>    
> 2. 📷 **提供多样化训练照片**：
>    - 拍摄 8-15 张不同角度（正脸、左右侧）
>    - 不同光线（明亮、昏暗）
>    - 不同状态（戴眼镜/不戴）
>    
> 3. 🧪 **测试验证**：
>    - 学习完成后在实际环境测试
>    - 观察调试信息中的置信度数值
>    - 根据误识别情况调整阈值
> 
> **技术限制：** 这是当前轻量级人脸识别库的通用问题，并非本应用特有缺陷。如需更高准确度，建议配合"仅识别正脸"功能使用，可显著降低误报率。

### 典型使用场景

**场景1：防止他人偷看屏幕（基本模式）**
- 设置人脸阈值为 2
- 处理方式：仅调节亮度
- 当只有你自己时（1张人脸）：保持正常亮度
- 当有人靠近（≥2张人脸）：自动降低亮度保护隐私

**场景2：通知提醒（隐蔽模式）**
- 处理方式：仅系统通知
- 通知内容：自定义为"有新消息"等
- 检测到有人靠近时收到通知，但不改变亮度
- 适合不想打扰工作但需要提醒的场景

**场景3：高级识别（精准模式）**
1. 使用照片采集工具（Ctrl+T / Cmd+T）拍摄5-10张自己的照片
2. 在高级设置中学习分析"本人"类别
3. 识别模式选择"除本人外"
4. 只有检测到非本人才触发（更精准）

**场景4：正脸识别（减少误触发）**
- 开启"仅识别正脸"选项
- 正脸严格度：适中（推荐）
- 只有正面看屏幕才触发，侧脸路过不触发
- 视觉反馈：正脸显示绿框，侧脸显示橙框并标记"侧脸"
- 适合办公环境，减少他人路过时的误报

### ⚠️ 正脸识别与人脸阈值配合使用（重要）

**核心逻辑：** 开启"仅识别正脸"后，**侧脸不会被计入人脸数量**，这会影响触发判断。

#### 场景分析

**场景A：摄像头里只有你 + 偷窥者**

| 配置 | 检测结果 | 人脸阈值建议 | 说明 |
|------|----------|--------------|------|
| **不开启正脸识别** | 本人（1）+ 偷窥者（1）= 2张 | **阈值设为 2** | ≥2张人脸时触发 ✅ |
| **开启正脸识别<br>本人永远侧脸** | 本人侧脸（不计入）+ 偷窥者正脸（1）= 1张 | **阈值设为 1** | ≥1张人脸时触发 ✅ |
| **开启正脸识别<br>本人有时正脸** | 有时本人也被计入 | ⚠️ **阈值无法满足需求** | 本人正脸时误触发 ❌ |

**解决方案：**

1. **情况1：能保证自己永远侧脸**（如摄像头在侧面）
   ```
   ✅ 基本设置：
      - 开启"仅识别正脸"
      - 人脸阈值改为 1（而不是2）
      - 不需要高级设置
   ```

2. **情况2：无法保证自己侧脸**（如摄像头在正前方）
   ```
   ✅ 必须配合高级设置：
      - 开启"仅识别正脸"
      - 学习"本人"照片
      - 识别模式：除本人外
      - 此时人脸阈值已失效，由高级识别控制
   ```

#### 实际示例

**示例1：侧面摄像头（推荐配置）**
- 摄像头位置：笔记本左侧/右侧
- 你的角度：永远侧脸
- 偷窥者：从背后正脸看屏幕
- 配置：正脸识别 + 阈值1 ✅

**示例2：正前方摄像头（需要高级模式）**
- 摄像头位置：笔记本屏幕上方
- 你的角度：正脸工作
- 偷窥者：从侧面正脸靠近
- 配置：正脸识别 + 高级识别（除本人外）✅

#### 判断自己的摄像头角度

启动应用后观察调试信息：
- 显示"➡️侧脸" → 可以用阈值1
- 显示"正脸" → 必须用高级识别

**总结：** 正脸识别功能很强大，但需要根据摄像头位置选择合适的配合策略。如果摄像头角度导致自己也是正脸，单纯的人脸数量阈值就无法满足需求，必须启用高级识别功能。

## 🔧 技术亮点

### 亮度控制实现

#### Windows
本项目**不使用**已弃用的 `wmic` 命令，而是直接调用 **PowerShell + WMI** API：

```powershell
# 获取亮度
powershell -Command "(Get-WmiObject -Namespace root/WMI -Class WmiMonitorBrightness).CurrentBrightness"

# 设置亮度
powershell -Command "(Get-WmiObject -Namespace root/WMI -Class WmiMonitorBrightnessMethods).WmiSetBrightness(1, 50)"
```

优势：
- ✅ 无需第三方依赖
- ✅ 打包后正常工作
- ✅ 不依赖已弃用的 wmic
- ✅ 兼容 Windows 10/11

#### macOS
使用 **DisplayServices 私有框架**，通过编译的 Objective-C 二进制程序控制亮度：

```objectivec
// 动态加载 DisplayServices 框架
void* handle = dlopen("/System/Library/PrivateFrameworks/DisplayServices.framework/DisplayServices", RTLD_LAZY);

// 调用亮度控制函数
DisplayServicesSetBrightness(display, brightness);
DisplayServicesGetBrightness(display, &brightness);
```

优势：
- ✅ 无需用户安装任何依赖
- ✅ 直接控制 MacBook 内置屏幕
- ✅ 打包后正常工作
- ✅ 不需要辅助功能权限

### 性能优化建议

#### 基本模式 vs 高级模式

| 模式 | CPU 占用 | 推荐检测间隔 | 适用场景 |
|------|----------|--------------|----------|
| **基本模式** | 低 | 500ms（默认） | 仅检测人脸数量，不做识别 |
| **高级模式** | 较高 | **1000ms（推荐）** | 需要人脸识别（本人/可信任/不可信任） |
| **高级+正脸识别** | 最高 | **1000ms（强烈推荐）** | 人脸识别 + 正脸角度判断 |

#### 优化建议

**启用高级设置时：**
1. ⚠️ **调整检测间隔**：将 `500ms` 改为 `1000ms`（1秒）
   - 理由：高级模式需要对每个人脸进行 128 维特征提取和欧氏距离计算



#### macOS 架构支持

- ⚠️ **Intel (x86_64)**: 理论支持，**未在真机测试**
  - Universal 二进制包含 Intel 支持

### 人脸识别功能限制

⚠️ **高级识别模式使用须知**：

**核心问题：** 本应用使用轻量级人脸识别库（face-api.js），虽然适合在浏览器中运行，但准确度存在客观局限：

1. **误识别风险**：
   - 相似度高的陌生人可能被误识别为"本人"或"可信任"
   - 同一人在不同光线/角度下可能被识别为不同人
   - 双胞胎、亲属等相似面孔难以区分

2. **环境敏感性**：
   - 光线变化（逆光、暗光）会显著影响准确度
   - 遮挡（口罩、手、头发）可能导致识别失败
   - 摄像头角度、距离变化影响特征提取

3. **训练数据依赖**：
   - 照片数量少（<5张）：识别不稳定
   - 照片多样性不足：新环境下失效
   - 照片质量差：模糊、过曝会降低效果

**强烈建议：**
- 🔧 **调低识别阈值**：从默认 `0.5` 降至 `0.4` 或 `0.35`，提高识别严格度
- 📸 **增加训练照片**：每类至少 8-15 张，涵盖不同角度、光线、表情
- 🧪 **实际测试验证**：学习完成后在真实使用环境中充分测试
- 👁️ **配合正脸识别**：开启"仅识别正脸"可显著降低误报率
- 💡 **保守使用模式**：初期建议使用"除本人外"模式，而非"仅针对不可信目标"

**替代方案：** 如果高级识别效果不理想，**基本模式**（仅检测人脸数量）仍然是可靠的防窥屏方案。

### 性能优化建议

- 使用 TinyFaceDetector 模型可提升性能
- 增加检测间隔（如 1000ms）可降低 CPU 占用
- 关闭"显示人脸检测框"可提升性能
